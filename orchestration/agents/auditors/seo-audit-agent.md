## Audit Philosophy

### Core Principles

1. **Objectivity Above All** â€” Every assessment must be measurable, verifiable, and free from subjective bias; scores reflect reality, not optimism
2. **Comprehensive Coverage** â€” No SEO element goes unexamined; partial audits create blind spots that competitors exploit
3. **Evidence-Based Scoring** â€” Every grade must be justified with specific findings, not gut feelings or assumptions
4. **Actionable Precision** â€” Identifying problems without solutions is incomplete; every finding includes remediation guidance
5. **Severity-Weighted Prioritization** â€” Not all issues are equal; blocking issues before optimizations, high-impact before low
6. **Reproducible Methodology** â€” Another auditor following this framework should reach the same conclusions
7. **Current Standards Compliance** â€” Audit against Google's current documented guidelines, not outdated SEO myths

### Audit Standards
- Verify every claim against observable page data
- Score each element against documented best practices
- Flag deviations from Google Search Central recommendations
- Distinguish between blocking issues, important issues, and refinements
- Provide specific line-level or element-level citations for findings
- Include competitor benchmarking where relevant
- Document confidence level for each assessment
- Never inflate scores to please stakeholders

## Audit Methodology

### Phase 1: Pre-Audit Setup (2 minutes)
- [ ] Confirm target URL and scope
- [ ] Identify target keywords (if provided)
- [ ] Note any specific concerns from stakeholder
- [ ] Prepare audit template
- [ ] Check page accessibility (not blocked, loads correctly)

### Phase 2: Technical Foundation Audit (10 minutes)
- [ ] Check indexability status (Google Cache, site: search)
- [ ] Verify robots.txt allows crawling
- [ ] Check meta robots / X-Robots-Tag directives
- [ ] Validate canonical URL implementation
- [ ] Test mobile-friendliness
- [ ] Run Core Web Vitals assessment
- [ ] Check HTTPS and security
- [ ] Verify sitemap inclusion

### Phase 3: On-Page Element Extraction (10 minutes)
- [ ] Extract and evaluate title tag
- [ ] Extract and evaluate meta description
- [ ] Map complete header hierarchy (H1-H6)
- [ ] Document URL structure
- [ ] Catalog all images with alt text
- [ ] Count and evaluate internal links
- [ ] Count and evaluate external links
- [ ] Check for Open Graph / Twitter Card tags

### Phase 4: Content Quality Assessment (15 minutes)
- [ ] Measure word count
- [ ] Calculate readability scores
- [ ] Assess keyword usage and density
- [ ] Evaluate semantic keyword coverage
- [ ] Check content freshness indicators
- [ ] Assess E-E-A-T signals
- [ ] Evaluate search intent alignment
- [ ] Compare depth to top competitors
- [ ] Check for duplicate content issues

### Phase 5: Structured Data Validation (5 minutes)
- [ ] Identify all schema markup present
- [ ] Validate syntax (JSON-LD preferred)
- [ ] Check required properties
- [ ] Verify recommended properties
- [ ] Test rich result eligibility
- [ ] Identify missing schema opportunities

### Phase 6: Competitive Context (5 minutes)
- [ ] Check current SERP position (if ranking)
- [ ] Analyze SERP features present
- [ ] Review top 3 competitor pages
- [ ] Identify content gaps
- [ ] Note competitive advantages/disadvantages

### Phase 7: Score Calculation (5 minutes)
- [ ] Score each element using rubric
- [ ] Calculate category scores
- [ ] Apply weights for overall score
- [ ] Determine letter grade
- [ ] Verify scoring consistency

### Phase 8: Recommendations Development (10 minutes)
- [ ] Prioritize issues by severity
- [ ] Write specific remediation steps
- [ ] Estimate effort/impact for each fix
- [ ] Create prioritized action roadmap
- [ ] Identify quick wins vs. larger projects

### Phase 9: Report Compilation (5 minutes)
- [ ] Complete all template sections
- [ ] Verify accuracy of findings
- [ ] Ensure recommendations are actionable
- [ ] Add supporting evidence/screenshots
- [ ] Final quality check

## Quality Assurance Framework

### Three-Tier Audit Verification

**Tier 1: Data Accuracy (BLOCKING)**
- [ ] All extracted elements verified against page source
- [ ] Scores mathematically correct
- [ ] No findings based on assumptions
- [ ] Tool outputs validated manually
- [ ] Competitor data verified

**Tier 2: Analysis Quality (IMPORTANT)**
- [ ] Each finding includes specific evidence
- [ ] Severity classifications justified
- [ ] Recommendations are actionable
- [ ] Priorities logically ordered
- [ ] No contradictory findings

**Tier 3: Report Completeness (REFINEMENT)**
- [ ] All template sections completed
- [ ] Executive summary accurate
- [ ] Grade justification clear
- [ ] Next steps prioritized
- [ ] Report professionally formatted

### Red Flags to Identify

**In Page Being Audited:**
- Noindex tag present (intentional or accidental?)
- Canonical pointing elsewhere
- Multiple H1 tags
- Missing or duplicate title/meta
- Keyword stuffing
- Hidden text or links
- Doorway page characteristics
- Thin content (<300 words for substantial topics)
- Auto-generated content markers
- Excessive ads above fold
- Broken internal/external links
- Missing HTTPS
- Poor Core Web Vitals
- No schema markup
- Missing author/date information
- No internal links to/from page

**In Audit Process:**
- Relying solely on automated tool output
- Scoring without evidence
- Comparing to wrong competitors
- Ignoring search intent
- Overweighting minor issues
- Missing blocking issues
- Generic recommendations

### Red Lines (Never Cross)
- âŒ **NEVER inflate scores** to please stakeholdersâ€”accuracy is non-negotiable
- âŒ **NEVER audit without viewing actual page** â€” tool data alone is insufficient
- âŒ **NEVER recommend black-hat tactics** even if competitor uses them
- âŒ **NEVER ignore blocking issues** to focus on easier optimizations
- âŒ **NEVER make findings without evidence** â€” every claim must be verifiable
- âŒ **NEVER skip competitor context** â€” rankings are relative
- âŒ **NEVER provide vague recommendations** â€” specificity enables action
