# Invocation Templates

Comprehensive, structured templates for invoking Esy agents with maximum context and quality output.

## Purpose

These templates ensure:
- **Complete context** — Agents receive all information needed for optimal output
- **Consistent quality** — Standardized inputs produce predictable excellence
- **Uniqueness enforcement** — Explicit protocols prevent repetition of existing work
- **Intelligent defaults** — Gaps are filled automatically when you want to move fast
- **Award-winning output** — Technical and creative standards are explicitly defined

## How to Use Templates

Simply `@` the template with your idea:

```
Using @agents/InvocationTemplates/visual-essay-creation.md,
create a visual essay about [YOUR TOPIC]
```

### Operating Modes

| Mode | How to Trigger | What Happens |
|------|----------------|--------------|
| **Quick** | Add "just run with it" or "fill in the gaps" | Verifies uniqueness, applies intelligent defaults, proceeds immediately |
| **Guided** | Add "guide me through" or "ask me questions" | Asks all strategic questions interactively |
| **Auto** | Just provide topic (default) | Asks 3-5 key questions, fills rest with defaults |

### Examples

```
# Quick — minimal friction, maximum speed
Using @agents/InvocationTemplates/visual-essay-creation.md,
create a visual essay about the invention of glass. Just run with it.

# Guided — full control over every decision
Using @agents/InvocationTemplates/visual-essay-creation.md,
create a visual essay about the invention of glass. Guide me through all options.

# Auto — balanced (default behavior)
Using @agents/InvocationTemplates/visual-essay-creation.md,
create a visual essay about the invention of glass.
```

## Template Structure Standard

Every invocation template follows this structure:

```
1. TEMPLATE METADATA
   - Purpose, target agents, quality standards

2. UNIQUENESS PROTOCOL
   - How to verify this doesn't replicate existing work

3. PRE-FLIGHT QUESTIONNAIRE
   - Strategic questions to maximize output quality

4. CONTENT ARCHITECTURE
   - How to structure the output (narrative, visual, technical)

5. TECHNICAL REQUIREMENTS
   - Engineering standards, performance budgets

6. RESEARCH REQUIREMENTS
   - Depth, sources, verification standards

7. INVOCATION TEMPLATE
   - The actual prompt structure to use

8. POST-CREATION VERIFICATION
   - How to verify output meets standards
```
